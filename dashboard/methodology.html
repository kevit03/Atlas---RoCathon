Method

A short explanation of the score, the weights, and the research behind the demo.

Scoring stack

Core formulas

Challenge engine

Official ranker

projected_normalized = (projected_score - 60) / 40

final_score =
100 * (
  alpha * semantic_score +
  (1 - alpha) * projected_normalized
)

This is the official challenge formula: normalized semantics plus normalized projected value.

Longer explanation

The ranker uses a convex combination because projected score already carries meaningful business signal. Once both components are normalized, linear fusion preserves that magnitude instead of collapsing the decision into rank positions.

Dashboard screening

Atlas screen

semantic_proxy = 0.55 * industry_match
               + 0.45 * query_overlap

commercial_index = 0.65 * projected_normalized
                 + 0.20 * engagement_norm
                 + 0.15 * gmv_norm

atlas_score =
100 * (
  0.38 * industry_match +
  0.27 * query_overlap +
  0.15 * audience_fit +
  0.20 * commercial_index
)

Atlas is the live demo score: stricter on relevance than on scale so the shortlist stays on-brief.

Longer explanation

Atlas is meant for exploration across the whole creator universe, not just the challenge output. That is why profile fit receives the largest weight and the commercial block is capped so pure size does not overwhelm contextual fit.

Quiz advisor

Advisor overlay

advisor_score =
  atlas_score
  + objective_bias
  + persona_bias
  + posture_bias

objective_bias favors:
- GMV for sales
- semantic fit for loyalty
- reach and views for awareness

The advisor adds a light campaign bias on top of the base score.

Longer explanation

The quiz is not a second ranking system. It lightly tilts the shortlist based on campaign objective, voice, and rollout posture so the output feels closer to a planning recommendation than a static leaderboard.
